# 视觉伺服控制系统改进说明

## 主要问题分析与解决方案

### 1. 深度图问题（已修复）
**问题**：原代码使用固定值0.5的虚拟深度图，无法反映真实场景深度
```python
# 原代码（错误）
depth = np.ones((rgb_image.shape[0], rgb_image.shape[1]), dtype=np.float32) * 0.5
```

**解决方案**：
- 使用MuJoCo的真实深度渲染功能
- 正确转换深度图格式（归一化深度 → 真实深度）
- 添加深度有效性检查

```python
# 新代码（正确）
depth_image = renderer.render(depth=True)
depth = mjr_znear * mjr_zfar / (mjr_zfar - depth_image * (mjr_zfar - mjr_znear))
```

### 2. 视觉伺服控制算法改进

#### 控制参数优化
- **降低比例增益**：从-0.6降至0.3，提高系统稳定性
- **添加PID控制**：引入积分和微分项，减少稳态误差
- **速度限制**：添加安全速度限制（0.3 rad/s）

#### 滤波系统
- **深度滤波**：使用指数移动平均滤波（α=0.7）
- **位置滤波**：对检测框坐标进行平滑处理（α=0.8）
- **减少噪声影响**：提高控制精度

#### 雅可比矩阵改进
- **更准确的相机内参**：使用RealSense D435i标准参数
- **四角点控制**：使用目标框四个角点进行更精确的控制
- **归一化坐标**：正确转换像素坐标到归一化坐标

### 3. 相机标定参数修复

**原参数**：
```python
f = 400  # 不准确的焦距
```

**新参数**：
```python
fx = 615.0  # 焦距x（基于RealSense D435i）
fy = 615.0  # 焦距y
cx = 320.0  # 主点x
cy = 240.0  # 主点y
```

### 4. 系统鲁棒性增强

#### 目标丢失处理
- **丢失计数**：跟踪连续丢失帧数
- **紧急停止**：超过阈值（10帧）自动停止
- **状态显示**：实时显示目标丢失状态

#### 安全机制
- **控制误差检查**：超过阈值（0.1）停止控制
- **速度限制**：防止机器人运动过快
- **矩阵求逆保护**：处理奇异矩阵情况

#### 手动控制
- **'r'键**：重置紧急停止状态
- **'e'键**：手动触发紧急停止
- **'q'键**：退出程序

### 5. 性能优化

#### YOLO推理优化
- **帧跳跃**：每2帧进行一次检测，减少计算负载
- **置信度阈值**：提高到0.5，减少误检
- **模型预热**：启动时预热模型，提高首次推理速度

#### 线程同步
- **检测锁**：保护检测数据的线程安全
- **控制锁**：保护控制计算的线程安全

## 使用建议

### 1. 参数调优
根据实际机器人性能调整以下参数：
- `Kp, Ki, Kd`：PID控制参数
- `alpha_depth, alpha_xywh`：滤波系数
- `safety_velocity_limit`：安全速度限制

### 2. 相机标定
建议使用提供的`camera_calibration.py`进行实际相机标定，获得更准确的内参。

### 3. 监控调试
- 观察控制台输出的误差信息
- 监控目标丢失计数
- 注意紧急停止状态

## 键盘控制说明

- **'q'**：退出程序
- **'r'**：重置紧急停止（仅在紧急停止状态下有效）
- **'e'**：手动触发紧急停止
